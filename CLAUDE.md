# InDepth Analysis — Research Report Platform

## Project Overview
Full-spectrum investment research platform with deep analysis report generation, reference document management, semantic search, and a pluggable skills system. Python 3.12, uv.

## Pending Tasks
See `~/projects/orchestrator/tasks/indepth_analysis.md` for journal-tracked tasks assigned to this project.
Auto-generated by the orchestrator journal system on each publish.

## Architecture
- `src/indepth_analysis/` — main package (src layout)
  - `cli.py` — argparse CLI entry point
  - `config.py` — AnalysisConfig, ReferenceConfig
  - `db.py` — ReferenceDB (sources, reports, chunks, embeddings)
  - `analysis/` — analysis engines
    - `fundamental.py`, `technical.py`, `macro.py` — core analysis dimensions
    - `sentiment.py`, `options_flow.py` — market sentiment and flow analysis
    - `portfolio.py` — portfolio context integration
    - `aggregator.py` — multi-dimension synthesis
  - `data/` — data clients
    - `kcif_client.py` — KCIF (Korea Center for International Finance) report scraper and downloader
    - `market_data.py` — market data abstraction
    - `yfinance_client.py`, `ibkr_client.py` — equity data sources
    - `ecb_client.py`, `eurostat_client.py` — European macro data
    - `sheets_client.py` — Google Sheets portfolio context
  - `models/` — InvestmentReport, ReportData, reference models
  - `output/` — report renderers
    - `ReportRenderer` (Rich terminal), `MarkdownRenderer`, `charts`
    - `notion_publisher.py` — Notion publishing
    - `slide_renderer.py` — PowerPoint slide generation
  - `processing/` — PDF extraction, chunking, embedding pipeline
  - `search/` — semantic retriever over reference documents
  - `skills/` — pluggable report generators
    - `base.py` — base skill interface
    - `euro_macro/` — European macro monthly investment report
      - KCIF agent: scrapes and reads KCIF PDF reports, extracts structured findings
      - Synthesis: creates investment-oriented narrative from findings (Claude API)
      - Slide generation: optional PowerPoint output
- `references/` — downloaded reference PDFs (KCIF reports, etc.)
- `reports/` — generated analysis reports (scanned by orchestrator journal system, mtime-based recursive)
- `tests/` — pytest test suite

## Commands
- `uv run python -m indepth_analysis.cli analyze <TICKER>` — full investment analysis
- `uv run python -m indepth_analysis.cli publish <report.md>` — publish to Notion
- `uv run python -m indepth_analysis.cli publish <report.md> --attach slide.pptx` — with slide attachment
- `uv run python -m indepth_analysis.cli update kcif` — scrape and download latest KCIF reports
- `uv run python -m indepth_analysis.cli process` — extract text, chunk, embed reference PDFs
- `uv run python -m indepth_analysis.cli search "query"` — semantic search over reference docs
- `uv run python -m indepth_analysis.cli status` — database status and cost summary
- `uv run python -m indepth_analysis.cli report euro-macro` — monthly European macro report
- `uv run python -m indepth_analysis.cli report euro-macro --collect-only` — KCIF agent phase only
- `uv run python -m indepth_analysis.cli report euro-macro --from-findings <path>` — synthesize from saved findings JSON
- `uv run python -m indepth_analysis.cli report euro-macro --slide` — include PowerPoint output

## Skills System
`skills/` provides a pluggable interface for specialized report types:
- **euro_macro**: Monthly European macro investment report
  - Phase 1: KCIF agent scrapes reports, reads PDFs, extracts findings → `reports/euro_macro/YYYY-MM-findings.json`
  - Phase 2: Synthesis creates investment-oriented narrative → `reports/euro_macro/YYYY-MM.md`
  - Phase 3 (optional): Slide generation → `reports/euro_macro/YYYY-MM.pptx`

## Data Sources
- **KCIF**: Korean Center for International Finance — macro reference reports (scraped + downloaded to references/KCIF/)
- **yfinance / IBKR**: Equity and market data for stock analysis
- **ECB / Eurostat**: European economic indicators (rates, GDP, inflation)
- **Google Sheets**: Portfolio context from TJAM

## Conventions
- Use uv for dependency management
- Format with ruff (line-length 88, double quotes)
- Lint rules: E, F, I, N, W, UP
- Tests in `tests/`, run with `uv run pytest`
- Python 3.12+ features encouraged
- Reports in `reports/` are scanned by orchestrator (mtime-based, recursive subdirectories)
