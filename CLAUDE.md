## Architecture
- `src/indepth_analysis/` — main package (src layout)
  - `cli.py` — argparse CLI entry point (`uv run indepth`)
  - `config.py` — AnalysisConfig, ReferenceConfig
  - `db.py` — ReferenceDB (sources, reports, chunks, embeddings)
  - `analysis/` — analysis engines
    - `fundamental.py`, `technical.py`, `macro.py` — core analysis dimensions
    - `sentiment.py`, `options_flow.py` — market sentiment and flow analysis
    - `portfolio.py` — portfolio context integration
    - `aggregator.py` — multi-dimension synthesis
  - `data/` — data clients
    - `kcif_client.py` — KCIF (Korea Center for International Finance) report scraper
    - `market_data.py` — market data abstraction
    - `yfinance_client.py`, `ibkr_client.py` — equity data sources
    - `ecb_client.py`, `eurostat_client.py` — European macro data
    - `sheets_client.py` — Google Sheets portfolio context
  - `models/` — InvestmentReport, ReportData, reference models, dev_welfare models
  - `output/` — report renderers
    - `notion_publisher.py` — Notion publishing (markdown → Notion blocks)
    - `slide_renderer.py` — PowerPoint slide generation
  - `processing/` — PDF extraction, chunking, embedding pipeline
  - `search/` — semantic retriever over reference documents
  - `skills/` — pluggable report generators
    - `base.py` — base skill interface
    - `euro_macro/` — **European macro monthly investment report** ← primary skill
    - `dev_welfare/` — International development & social welfare reports
    - `issue_track/` — Issue tracking report (experimental)
- `references/` — downloaded reference PDFs (KCIF reports, etc.)
- `reports/` — generated analysis reports (scanned by orchestrator journal system)
- `scripts/` — operational runbooks and helper scripts
- `tests/` — pytest test suite

## Euro Macro Pipeline (핵심 파이프라인)

월간 유럽 거시경제 보고서를 자동으로 생성하고 노션에 퍼블리시하는 파이프라인.

### 파이프라인 구성

```
[수집 단계] ──────────────────────────────────────────────────────────
  KCIFAgent          ─ KCIF DB 시맨틱 검색 (top_k=8, chunk=1500자)
  ForexFactoryAgent  ─ FF JSON feed (lastweek/thisweek/nextweek)
                       + FRED CB 금리 (USD/EUR/GBP/JPY/KRW)
                       + frankfurter.app FX 스냅샷 (EUR 기준)
                       + sigma alert 계산
  WebResearchAgent   ─ 10개 토픽 × Claude CLI 병렬 서브프로세스
                       (--allowedTools "WebSearch WebFetch", 동시 최대 4개)
                       토픽: ECB 통화정책, 유로존 통계, 금융시장, EU 정치,
                             무역/관세, 에너지/원자재, 재정/국채, 은행/신용,
                             유럽 정치지형(언론), 유럽 정치지형(연구기관)

[합성 단계] ──────────────────────────────────────────────────────────
  orchestrator._synthesize()
    ├─ ForexFactory LLM digest → context 앞에 prepend
    ├─ Claude CLI: claude-opus-4-20250514, timeout=600s
    ├─ ENHANCED_SYNTHESIS_SYSTEM_PROMPT (섹션당 600자+, 수치 3개+, 인라인 출처)
    └─ 본문 8개 섹션 파싱

[결정론적 섹션] ──────────────────────────────────────────────────────
  MacroSectionsBuilder
    ├─ Section A: ForexFactory 향후 14일 캘린더 표
    ├─ Section B: 주요 지표 서프라이즈 (최근 30일, ▲/▼)
    ├─ Section G: FX 환율 스냅샷 (EUR 기준)
    └─ Section H: 중앙은행 정책금리 표 (데이터 있을 때)

[부록 단계] ──────────────────────────────────────────────────────────
  AppendixBuilder
    ├─ 부록 I:  언론 보도 요약 (WebResearch 카테고리별, 결정론적)
    ├─ 부록 II: 주요국별 심층 분석 DE/FR/IT/ES (Opus LLM)
    ├─ 부록 III: 리스크 시나리오 Base/Bear/Bull (Opus LLM)
    ├─ 부록 IV: 출처 목록 (결정론적)
    └─ 부록 V:  유럽 정치지형 심층 분석 (Opus LLM)
                타임라인 표 / 현황 / 전망 / 거시·자본시장 영향
                언론·연구기관 논평 / 참고문헌 표 (제목·URL·요약)
```

## 월간 런북 (Monthly Runbook)

### 전제 조건 확인

```bash
# 환경 변수 (.env)
NOTION_TOKEN=...
NOTION_PAGE_ID_INDEPTH_ANALYSIS=...

# KCIF DB 최신화 (월 1회, 보고서 생성 전)
uv run indepth update kcif
uv run indepth process  # PDF → 청크 → 임베딩

# claude CLI 확인
which claude  # PATH에 있어야 함
```

### 월간 보고서 생성 (표준 실행)

```bash
# 당월 보고서 생성 (현재 연월 자동 사용)
uv run indepth report euro-macro

# 특정 연월 지정
uv run indepth report euro-macro --year 2026 --month 5

# 옵션
#   --no-web          WebResearch 건너뜀 (오프라인 / 빠른 테스트)
#   --no-macro        ForexFactory 건너뜀 (오프라인)
#   --force-refresh   ForexFactory 4시간 캐시 무시
#   --collect-only    수집만 하고 findings JSON 저장 (합성 건너뜀)
#   --from-findings   기존 findings JSON으로 재합성 (재실행 빠름)
#   --slide           PowerPoint 슬라이드 함께 생성
```

### 단계별 실행 (긴 파이프라인 분리 실행)

```bash
# 1단계: 수집만 (findings JSON 저장)
uv run indepth report euro-macro --year 2026 --month 5 --collect-only
# → reports/euro_macro/2026-05-findings.json

# 2단계: 저장된 findings로 합성 (재실행 시 빠름)
uv run indepth report euro-macro \
  --from-findings reports/euro_macro/2026-05-findings.json
# → reports/euro_macro/2026-05.md
```

### 노션 퍼블리시

```bash
uv run indepth publish reports/euro_macro/2026-05.md
# 슬라이드 첨부 시:
uv run indepth publish reports/euro_macro/2026-05.md \
  --attach reports/euro_macro/2026-05.pptx
```

### 스크립트 실행 (원스텝)

```bash
./scripts/monthly_euro_macro.sh          # 당월
./scripts/monthly_euro_macro.sh 2026 5   # 특정 연월
```

## 주요 장애 대응

| 증상 | 원인 | 대응 |
|---|---|---|
| WebResearch 토픽 타임아웃 | Claude CLI 느림 또는 claude 미설치 | `--no-web` 플래그 사용, 또는 `timeout_per_topic` 증가 |
| ForexFactory 404/WAF 차단 | JSON feed 변경 또는 Cloudflare 차단 | `--no-macro` 또는 `--force-refresh`, HTML backfill 시 브라우저 쿠키 전달 |
| KCIF 수집 0건 | DB 비어 있음 | `uv run indepth update kcif && uv run indepth process` |
| 합성 본문 비어 있음 | Claude CLI stream-json 파싱 실패 | `--verbose` 로그 확인, `claude -p ... --output-format stream-json --verbose` 직접 테스트 |
| 부록 LLM 호출 실패 | non-fatal, 로그 경고 | 부록 섹션 없이 계속 진행됨 (정상) |
| Notion 퍼블리시 실패 | 토큰 만료 또는 parent_id 오류 | `.env` 확인, Notion integration 권한 확인 |

## Conventions
- `uv run indepth` — CLI 진입점 (절대 `python -m` 사용 금지)
- Python 3.12+, uv 패키지 관리
- Anthropic SDK 사용 금지 — Claude는 `claude` CLI 서브프로세스로만 호출
- Format: ruff (line-length 88, double quotes)
- Lint: E, F, I, N, W, UP
- Tests: `uv run pytest`
- `stream-json` 파싱 패턴: `type=="assistant"` content blocks → text 수집, fallback `type=="result" subtype=="success"`

## Claude CLI Subprocess Notes
- Do NOT use `<system>` tags in prompts passed to `claude -p` — causes empty responses
- Always include `--output-format stream-json --verbose` together (stream-json requires verbose)
- WebResearch subprocess: add `--allowedTools "WebSearch WebFetch"`
- Synthesis subprocess: no extra tools (pure text generation)
