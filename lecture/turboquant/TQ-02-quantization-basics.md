## 1. 양자화란 무엇인가?

### 핵심 질문

> "실수 0.314159를 정수로 바꾸면 0이 됩니다. 잃어버린 0.314159는 어디로 갔나요?"

**양자화(Quantization)** = 연속적인 값을 **유한 개의 이산적 값**으로 변환하는 것.

### 일상의 양자화

| 원본 | 양자화 | 잃어버린 것 |
|------|--------|-----------|
| 체온 36.7°C | "정상" | 0.7°C의 차이 |
| 시험 점수 87.3 | "B+" | A-와의 경계 정보 |
| RGB (127, 63, 191) | "보라색" | 정확한 색조 |

### 컴퓨터에서의 양자화

```
FP32 (32비트): 0.314159265358979...  ← 높은 정밀도
FP16 (16비트): 0.3142               ← 중간 정밀도
INT8 (8비트):  0.31                  ← 낮은 정밀도
INT4 (4비트):  0.3                   ← 매우 낮은 정밀도
INT3 (3비트):  0                     ← 극도로 낮은 정밀도 (!)
```

KV Cache의 각 값이 **FP16 (16비트)**로 저장되어 있습니다. 이것을 **3비트**로 줄이려면, 16비트 중 **13비트를 버려야** 합니다. 과연 가능할까요?

## 2. 스칼라 양자화 (Scalar Quantization)

### 가장 단순한 양자화: 균등 양자화

실수 $x \in [a, b]$를 $B$ 비트로 양자화:

$$Q(x) = a + \frac{b-a}{2^B - 1} \cdot \text{round}\left(\frac{x - a}{b - a} \cdot (2^B - 1)\right)$$

$B = 3$ 비트이면 $2^3 = 8$개의 값만 표현 가능:

```
원래 범위 [-1, 1]을 3비트로 양자화:

  -1.0   -0.71  -0.43  -0.14   0.14   0.43   0.71   1.0
   ●──────●──────●──────●──────●──────●──────●──────●
   000    001    010    011    100    101    110    111

0.5는 → 101 (= 0.43)로 매핑.  오차 = 0.07
```

### 양자화 오차 (Quantization Error)

$$e = x - Q(x)$$

이 오차의 크기를 **왜곡(Distortion)**이라 합니다.

---

## 3. 왜곡을 측정하는 두 가지 방법

TurboQuant가 해결하려는 핵심 문제는 **두 가지 왜곡을 동시에 최소화**하는 것입니다.

### 3.1 MSE 왜곡 (Mean Squared Error)

$$D_{\text{MSE}} = E\left[\|x - Q(x)\|^2\right]$$

"원본과 양자화된 값이 얼마나 가까운가?"

- KV Cache의 **Value** 벡터에 중요
- Value를 잘못 양자화하면 → 출력 텍스트의 품질이 직접 저하

### 3.2 내적 왜곡 (Inner Product Distortion)

$$D_{\text{IP}} = E\left[\left(\langle x, y \rangle - \langle Q(x), Q(y) \rangle\right)^2\right]$$

"두 벡터의 내적이 양자화 전후에 얼마나 달라지는가?"

- KV Cache의 **Key** 벡터에 중요
- Attention score = $QK^T$ = **Query와 Key의 내적**
- Key를 잘못 양자화하면 → **엉뚱한 토큰에 attention**

### 왜 두 가지를 동시에 다뤄야 하는가?

| 양자화 대상 | 필요한 품질 | 왜? |
|-----------|-----------|-----|
| **Key** 벡터 | 내적 보존 | Attention score = $QK^T$ |
| **Value** 벡터 | MSE 최소화 | 출력 = attention 가중 Value 합 |

기존 방법의 문제: MSE를 최소화하는 양자화는 내적에 **체계적 편향(bias)**을 만듦.

---

## 4. 편향 (Bias) 문제 — 왜 이것이 치명적인가?

### 비편향 추정 vs 편향 추정

양자화기 $Q$가 내적의 **비편향 추정(unbiased estimator)**이려면:

$$E\left[\langle Q(x), Q(y) \rangle\right] = \langle x, y \rangle$$

**편향이 있으면** ($\neq$):

$$E\left[\langle Q(x), Q(y) \rangle\right] = \langle x, y \rangle + \text{bias}(x, y)$$

### Attention에서 편향의 재앙

$$\text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)$$

$QK^T$에 체계적 편향이 있으면:
- 특정 토큰들의 attention score가 **항상 높거나 낮게** 왜곡
- softmax가 이 왜곡을 **증폭** (exponential 함수이므로)
- 결과: 모델이 **엉뚱한 토큰에 집중** → 출력 품질 저하

### 구체적 예시

```
원래 attention scores: [0.1, 0.2, 0.5, 0.2]  → softmax → [0.17, 0.21, 0.36, 0.26]
                                                            ↑ 중요 토큰

편향 +0.3 추가:       [0.4, 0.5, 0.8, 0.5]  → softmax → [0.20, 0.22, 0.30, 0.28]
                                                            ↑ 중요도 희석!

→ 중요한 토큰(0.36 → 0.30)의 가중치가 줄어들고,
  덜 중요한 토큰(0.17 → 0.20)의 가중치가 늘어남
```

> **"모든 내적이 조금씩 편향되면, 모델 전체가 흐려집니다."**

## 5. 양자화 상수 (Quantization Constants)의 오버헤드

### 기존 방법의 숨겨진 비용

대부분의 양자화 기법은 각 블록마다 **min, max, scale** 값을 FP16으로 저장합니다:

```
블록 1: [q₁, q₂, ..., q₃₂] (3-bit × 32개 = 12 bytes)
        + min=FP16 (2 bytes) + max=FP16 (2 bytes) = 4 bytes 오버헤드!

실질 비트폭 = (12 + 4) / 32 × 8 = 4 bits/element
                                    ↑ 3비트가 아니라 사실상 4비트!
```

비트폭이 낮아질수록 이 오버헤드의 비중이 **폭발적으로** 증가:

| 목표 비트폭 | 블록 크기 32 | 실질 비트폭 | 오버헤드 비율 |
|-----------|------------|-----------|-------------|
| 8-bit | 8 + 1 = 9 | 9 bits | 12.5% |
| 4-bit | 4 + 1 = 5 | 5 bits | 25% |
| **3-bit** | **3 + 1 = 4** | **4 bits** | **33%!** |
| **2-bit** | **2 + 1 = 3** | **3 bits** | **50%!** |

> **3비트 이하에서는 양자화 상수 오버헤드가 양자화 자체보다 클 수 있습니다.**

### TurboQuant의 혁신: 오버헤드 제로

TurboQuant는 **양자화 상수를 전혀 저장하지 않습니다** (Data-Oblivious). 어떻게 가능한가? → 강의 4, 5에서 설명.

## 6. 벡터 양자화 (Vector Quantization)

### 스칼라 vs 벡터

| | 스칼라 양자화 | 벡터 양자화 |
|---|---|---|
| 단위 | 숫자 1개씩 | **벡터 전체를 한 번에** |
| 코드북 | $2^B$개 숫자 | $2^B$개 벡터 |
| 장점 | 단순 | **좌표 간 상관을 활용** → 더 낮은 왜곡 |
| 단점 | 좌표 간 상관 무시 | 코드북 설계가 어려움 |

### 왜 벡터 양자화가 더 나은가?

$d$차원 벡터 $x = (x_1, x_2, \ldots, x_d)$에서:

**스칼라**: 각 $x_i$를 독립적으로 양자화 → 좌표 간 관계 무시
**벡터**: 벡터 $x$ 전체를 코드북의 가장 가까운 벡터에 매핑

Shannon의 이론에 의하면, 벡터 양자화가 **항상** 스칼라 양자화보다 좋거나 같음:

$$D_{\text{vector}}^* \leq D_{\text{scalar}}^*$$

> **TurboQuant의 핵심 통찰**: 랜덤 회전을 적용하면, 좌표 간 상관이 사라져서 **스칼라 양자화로도 벡터 양자화에 근접하는 성능**을 달성할 수 있다.

## 8. 이 강의에서 기억할 것

1. **양자화** = 연속 값을 유한 이산 값으로 변환 (비트 수 줄이기)
2. **두 가지 왜곡**: MSE (값 자체의 오차)와 내적 왜곡 (Attention score 오차) — 둘 다 중요
3. **편향**이 있으면 softmax가 이를 증폭 → 모델 전체가 흐려짐
4. **양자화 상수 오버헤드**: 3비트 이하에서는 상수 저장 비용이 양자화 이득을 상쇄
5. TurboQuant의 목표: **MSE와 내적 왜곡을 동시에 최소화** + **오버헤드 제로**
